製作筆記：

這個專題是和同學一起完成的 AIoT 智慧路口號誌燈系統，整體由感測器、影像辨識、後端與前端等部分組成。
我主要負責號誌燈的模擬流程、資料庫的串接，以及統計圖表與前端介面設計。

這段經驗讓我學到，遇到限制時可以用替代方案先模擬流程，也體會到和組員協作、把各自完成的模組整合起來的重要性。

📘 API 練習

API（Application Programming Interface，應用程式介面）是一種「定義好的資料交換方式」。
前端程式可以透過 API 向後端伺服器發送請求（Request），伺服器則回傳資料（Response），常見格式是 JSON。
簡單來說，API 就是讓不同系統之間能以標準化方式溝通的橋樑。

如果把它想成「餐廳的菜單」：前端點什麼，後端就端回什麼資料。
在網頁裡，通常就是「用網址拿資料」，再把結果顯示出來。

🔍
API 基本觀念流程：Frontend → Backend → Response (JSON) → 回到 Frontend 顯示結果

程式碼截圖：

🔍
從回傳陣列取 data[0].url，塞進 <img> 顯示。按鈕可再次呼叫 API 換圖。

最小範例：

🔍
使用 fetch() 呼叫 https://api.thecatapi.com/v1/images/search，顯示隨機貓咪圖片 😸

學習心得：

  • 理解 API = 用網址拿資料，回來的多是 JSON；比想像中直覺。
  • 這個範例讓我把「呼叫 → 解析 → 顯示」串起來，跟我之前學的 JSON 很呼應。
  • 也了解到還有許多實用的 API，例如天氣、匯率等，未來可以嘗試結合到作品中。
